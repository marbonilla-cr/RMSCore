RMS — Tenant Bootstrap Data Loader

Deterministic Import System (Production-Ready v1)

⸻

Objective

Implement a Tenant Bootstrap Data Loader module that allows a tenant administrator to upload a Master Excel file containing the initial operational data required to start using the RMS.

The system must perform the following pipeline:

Excel Upload
↓
Excel Parser
↓
Column Alias Mapper
↓
Ledger Staging Tables
↓
Inline Data Editor
↓
Validation Engine
↓
Transactional Import
↓
System Check
↓
Tenant Ready

The goal is to enable rapid tenant bootstrap with minimal manual configuration.

This version must NOT use AI.

All parsing and column mapping must be deterministic using alias mappings.

⸻

Scope Restriction

The agent MUST NOT modify any RMS core modules.

The following areas are strictly forbidden to modify:
	•	POS
	•	Orders
	•	Inventory
	•	Payroll

⸻

Architectural Principles
	1.	Runs inside tenant scope only
	2.	Never writes directly to production tables
	3.	Uses staging ledger tables
	4.	Import must be fully transactional
	5.	Validation must happen before import
	6.	System check must run before tenant activation
	7.	Import must follow a strict dependency order

row_index INTEGER

created_at TIMESTAMP
updated_at TIMESTAMP

INDEX(session_id, sheet_name)

Responsibilities:

	•	Detect sheets

	•	Normalize headers using alias mapper

Validation Engine

Create file:

server/data-loader/validation-engine.ts

Endpoint:

POST /sessions/:id/validate

Validation rules:

Required fields must exist.

Referential validations:

product.category must exist in categories
product.tax must exist in taxes
modifier.group_name must exist in modifier_groups
product_modifiers.product_name must exist in products
product_modifiers.group_name must exist in modifier_groups

Validation result stored in staging rows.

Session status becomes:

validated


⸻

Import Engine

Create file:

server/data-loader/import-engine.ts

Endpoint:

POST /sessions/:id/import

Rules:
	•	Allowed only if session.status = validated
	•	Execute full DB transaction
	•	Follow strict import order

Pseudo process:

BEGIN

insert business
insert taxes
insert payment_methods
insert employees
insert categories
insert products
insert modifier_groups
insert modifiers
insert product_modifiers
insert tables
insert hr_config

COMMIT

On failure:

ROLLBACK
session.status = failed
