HU C1 — Void de pago debe revertir caja (cash_session) y no “reabrir orden”

CDA

	•	Si payment.method=cash: decrementar cash_session.expectedCash (y totals por método) por payment.amount_applied.
	•	NO cambiar order_item.status (cocina) ni “reabrir” ítems.
	•	Recalcular order.paid_amount y balance_due.
	•	Auditoría: PAYMENT_VOIDED con before/after.
DoD

HU C2 — División de pagos (pay-split) debe usar totales reales (incluye modifiers/discount/taxes)

	•	Calcular balance_due = total_amount - sum(payments PAID) usando recalcOrderTotal(orderId) (función única).
	•	Validar amount <= balance_due (tarjeta/SINPE) o permitir vuelto (cash).
	•	Prohibido calcular con productPriceSnapshot * qty.
DoD
	•	Orden con modifiers + descuento + impuesto servicio: saldo y pagos parciales cuadran.

HU C3 — Normalize-split debe recalcular con recalcOrderTotal()

CDA

	•	Llamar siempre recalcOrderTotal(orderId) para parent/child después de mover/normalizar.

HU C4 — Void payment NO debe afectar ítems de pagos válidos (multi payments)

HU C5 — Reopen order debe estar bloqueado o ser seguro (sin desalinear pagos)
