## T008 — Fix timezone: getToday() en dashboard

**Por qué primero:** Es una sola línea, cero riesgo, y es el bug que hace que el dashboard muestre cero ventas después de las 6pm hora CR.

**Archivo:** `client/src/pages/dashboard.tsx`

Buscar la función `getToday()`. Probablemente luce así:
```typescript
const getToday = () => new Date().toISOString().slice(0, 10);
```

`toISOString()` siempre retorna fecha UTC. A las 6:00 pm en Costa Rica ya es medianoche UTC del día siguiente, por eso el dashboard pide datos para "mañana" y no hay ventas.

Reemplazar con:
```typescript
const getToday = () =>
  new Date().toLocaleDateString("en-CA", { timeZone: "America/Costa_Rica" });
```

`"en-CA"` produce formato `YYYY-MM-DD`. Con el timezone de CR, a las 7pm el resultado sigue siendo "hoy".

Si la misma expresión `new Date().toISOString().slice(0, 10)` aparece en otros lugares del mismo archivo (por ejemplo en el `useState` inicial de la fecha), reemplazarla también.

**Criterio:** El dashboard muestra ventas del día correcto a cualquier hora CR.

## T005 — Backend: endpoint de items por proveedor

**Archivo:** El archivo que registra las rutas de inventario (buscar `registerInventoryRoutes` o el archivo con rutas `/api/inventory/...`)

**Primero verificar que la tabla `inv_supplier_items` existe:**
```sql
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'inv_supplier_items'
ORDER BY ordinal_position;
```

Si **no existe**, detenerse y reportar. No crearla — requiere revisión de schema.

Si **existe**, agregar este endpoint. **IMPORTANTE:** usar la ruta `/api/inventory/suppliers/:id/items` (con `inventory` completo, consistente con todas las demás rutas del módulo):

```typescript
app.get("/api/inventory/suppliers/:id/items", requireAuth, async (req, res) => {
  try {
    const supplierId = parseInt(req.params.id);
    if (isNaN(supplierId)) return res.status(400).json({ message: "ID inválido" });

    const mode = (req.query.mode as string) || "all";
    const categoriesParam = req.query.categories as string | undefined;
    const categories = categoriesParam
      ? categoriesParam.split(",").map((c: string) => c.trim()).filter(Boolean)
      : [];

    // Query principal
    let q = `
      SELECT
        ii.id,
        ii.sku,
        ii.name,
        ii.category,
        ii.base_uom,
        CAST(ii.on_hand_qty_base          AS FLOAT) AS on_hand_qty_base,
        CAST(ii.reorder_point_qty_base     AS FLOAT) AS reorder_point_qty_base,
        COALESCE(isi.purchase_uom, ii.base_uom)                    AS purchase_uom,
        CAST(COALESCE(isi.last_price_per_purchase_uom, 0) AS FLOAT) AS last_price,
        CAST(COALESCE(isi.units_per_purchase_uom, 1)      AS FLOAT) AS units_per_purchase_uom
      FROM inv_items ii
      JOIN inv_supplier_items isi ON isi.inv_item_id = ii.id
      WHERE isi.supplier_id = $1
        AND ii.active = true
    `;
    const params: any[] = [supplierId];
    let idx = 2;

    if (mode === "low_stock") {
      q += ` AND CAST(ii.on_hand_qty_base AS FLOAT) < CAST(ii.reorder_point_qty_base AS FLOAT)`;
    }
    if (categories.length > 0) {
      q += ` AND ii.category = ANY($${idx})`;
      params.push(categories);
      idx++;
    }
    q += ` ORDER BY ii.category NULLS LAST, ii.name`;

    // Categorías disponibles para este proveedor
    const catsQ = `
      SELECT DISTINCT ii.category
      FROM inv_items ii
      JOIN inv_supplier_items isi ON isi.inv_item_id = ii.id
      WHERE isi.supplier_id = $1
        AND ii.active = true
        AND ii.category IS NOT NULL
      ORDER BY ii.category
    `;

    const [itemsRes, catsRes] = await Promise.all([
      pool.query(q, params),
      pool.query(catsQ, [supplierId]),
    ]);

    res.json({
      items: itemsRes.rows,
      categories: catsRes.rows.map((r: any) => r.category),
    });
  } catch (err: any) {
    console.error("[supplier-items]", err.message);
    res.status(500).json({ message: err.message });
  }
});
```

**Criterio:** `GET /api/inventory/suppliers/1/items` retorna `{ items: [...], categories: [...] }`. `?mode=low_stock` filtra correctamente.
