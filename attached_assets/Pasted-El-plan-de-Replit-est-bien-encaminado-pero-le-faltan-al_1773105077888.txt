RMS — Tenant Bootstrap Data Loader

Deterministic Import System (v1)

Scope restriction

This implementation is strictly limited to the Data Loader module.

The agent must NOT modify any existing RMS core modules including:
	•	POS
	•	Orders
	•	Inventory
	•	Payroll
	•	KDS
	•	Reservations
	•	Loyalty
	•	Reporting

Only the following paths may be created or modified:

server/data-loader/*
migrations/0004_data_loader_staging.sql
client/src/pages/data-loader/*
client/src/components/data-loader/*

No other files may be changed.

⸻

Objective

Build a Tenant Bootstrap Data Loader module that allows a tenant administrator to upload a Master Excel file containing the initial operational data required to start using RMS.

The system must:
	1.	Accept an Excel upload
	2.	Parse sheets
	3.	Map columns using deterministic alias mapping
	4.	Load rows into staging ledger tables
	5.	Provide inline editing for verification
	6.	Validate referential integrity
	7.	Execute transactional import
	8.	Run automated system check
	9.	Mark tenant ready

This version must not use AI.

Parsing must be deterministic.

⸻

Architectural Principles
	1.	Runs inside tenant scope
	2.	Never writes directly to production tables
	3.	Uses staging ledger tables
	4.	Import must be transactional
	5.	Validation must happen before import
	6.	System check must run before activation
