T001: Clock-in sin horario requiere doble confirmación
FILES: server/routes.ts, client/src/pages/pin-login.tsx, client/src/pages/hr/mi-turno.tsx

Backend (server/routes.ts)
POST /api/auth/pin-clock (~630) y POST /api/hr/clock-in (~4909):
- Determinar si hay horario para hoy.
  “No hay horario” = no existe schedule, o isDayOff, o sin start/end válidos.
- Si NO hay horario y NO viene confirmNoSchedule=true:
  responder 200 { requireConfirm:true, message:"No tiene horario hoy. Ingrese su código nuevamente para confirmar." }
  NO crear punch.
- Si NO hay horario y viene confirmNoSchedule=true:
  clock-in normal + audit_event CLOCK_IN_NO_SCHEDULE_CONFIRMED.
- Si SÍ hay horario: clock-in normal.

Frontend
- Si requireConfirm=true: modal con message; al confirmar reenviar con confirmNoSchedule=true.

Acceptance
- Sin horario pide confirmación; con confirmación marca + audit; con horario funciona normal.

T003: Deducción automática almuerzo/café (>= 9h) sin tocar overtime
FILES: server/payroll.ts, client/src/pages/hr/reports.tsx

En computeDailyPayroll:
1) workedMinutes RAW como hoy.
2) overtimeMinutes:
   SOLO si clockOutType=MANUAL y scheduledEndAt existe y clockOutAt>scheduledEndAt:
     overtimeMinutes=max(0, clockOutAt - scheduledEndAt)
   sino 0
   (IMPORTANTE: scheduledEndAt debe ser datetime correcto del turno; si schedule guarda solo hora, construir datetime con fecha del día y si end<start entonces end=+1 día)
3) normalMinutesRaw = max(0, workedMinutes - overtimeMinutes)
4) unpaidBreakMinutes = (workedMinutes >= 540) ? 60 : 0
5) normalMinutesPaid = max(0, normalMinutesRaw - unpaidBreakMinutes)
6) overtimeMinutesPaid = overtimeMinutes
7) paidWorkedMinutes = normalMinutesPaid + overtimeMinutesPaid
8) Pagos:
   hourlyRate = dailyRate / Number(hrConfig.overtimeDailyThresholdHours || 8)
   multiplier = Number(hrConfig.overtimeMultiplier || 1.5)
   normalPay = round2((normalMinutesPaid/60)*hourlyRate)
   overtimePay = round2((overtimeMinutesPaid/60)*hourlyRate*multiplier)
9) Exponer unpaidBreakMinutes y paidWorkedMinutes en resultados.
10) computeRangePayroll acumula totalUnpaidBreakMin.
11) computeServiceForDay usa paidWorkedMinutes como peso (no raw).

UI reports.tsx:
- Columna "Desc." (min) en resumen y breakdown (HH:mm).
- Totales suman descuentos.

Acceptance
- >=540 => 60 desc del ordinario; <540 => 0; overtime intacto; UI muestra.
