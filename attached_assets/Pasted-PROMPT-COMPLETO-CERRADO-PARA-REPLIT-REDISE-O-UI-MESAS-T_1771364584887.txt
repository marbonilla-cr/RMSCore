1) Objetivo

Optimizar la pantalla donde el salonero toma y revisa órdenes de mesas, para:
	•	Agregar ítems más rápido sin depender del botón “+”.
	•	Ver más ítems por pantalla (orden grande = menos scroll).
	•	Notas por ítem sin campos visibles que consuman espacio.
	•	Dark Mode completo.
	•	Minimizar uso del teclado.
	•	Deshabilitar pull-to-refresh en todo el sistema.

2) Cambios UI/UX en MÓDULO MESAS / SALONERO

2.1 Layout por dispositivo

Móvil (teléfonos)
	•	Vista principal = Catálogo (categorías + grid productos).
	•	“Nueva Ronda” = Bottom Sheet (deslizable) para revisar/editar y enviar.
	•	Botón fijo para abrir ronda: 🧾 Ronda (X) (contador en tiempo real).
	•	Dentro del bottom sheet: botón sticky Enviar a Cocina.

Tablet / Desktop
	•	Layout 2 columnas:
	•	Izquierda: Categorías (lista vertical)
	•	Derecha: Productos (grid responsive)
	•	La ronda puede ser:
	•	panel lateral si hay espacio, o
	•	bottom sheet igual (aceptable).

⸻

2.2 Catálogo: categorías + productos (sin accordion)

Buscador sticky (teclado minimizado)
	•	El buscador NO debe mostrar teclado automáticamente.
	•	Por defecto mostrar solo un ícono 🔍 y placeholder tipo “Buscar”.
	•	Al tocar 🔍 se expande el input. Sin autofocus.
	•	Debounce 200ms.

Categorías

Reemplazar accordions por:

En móvil: chips horizontales (scroll) debajo del buscador
	•	categoría activa resaltada
	•	opcional: badge con conteo

En tablet/desktop: lista vertical de botones grandes
	•	categoría activa resaltada

Productos en grilla responsive (solo categoría seleccionada)

CSS Grid por ancho real (no por user agent):
	•	<= 360px: 2 columnas
	•	361–520px: 3 columnas
	•	521–820px: 4 columnas
	•	821–1100px: 5 columnas
	•	1100px: 6 columnas

Tarjeta de producto (compacta, tocable):
	•	Nombre grande (ellipsis)
	•	Precio pequeño
	•	Badge “Opciones” si tiene modificadores
	•	Si inactivo: NO se muestra
	•	Si sin porciones: disabled con “Agotado”

Acción tap:
	•	Si NO tiene modificadores requeridos: agrega 1 unidad a la ronda + toast “Agregado ✅”
	•	Si requiere modificadores: abrir modal/bottom sheet de modificadores (botones grandes)

Performance:
	•	Renderizar solo productos de la categoría activa (no todo el menú).
	•	Sin imágenes pesadas (si hay, lazy-load).

7) Deshabilitar pull-to-refresh (GLOBAL, todo el RMS)

Objetivo

Evitar refresco accidental por gesto hacia abajo en móviles. Solo refrescar con botón refresh.

Implementación requerida

7.1 CSS global
	•	html, body { overscroll-behavior-y: none; }
	•	Mantener scroll suave (no romper):
	•	ajustar -webkit-overflow-scrolling: touch; según corresponda

7.2 JS global (fallback)

Interceptar gesto swipe-down cuando el scroll está al tope:
	•	Si window.scrollY === 0 (o contenedor principal scrollTop === 0) y el gesto es hacia abajo → preventDefault()
	•	Usar { passive: false }

7.3 Contenedores con overflow

Si el scroll principal vive en un div (layout app):
	•	aplicar la lógica sobre ese contenedor, no solo window.

Reglas
	•	No bloquear scroll normal.
	•	No interferir con gesto lateral de back en iPhone.
	•	Refresh solo ocurre con el botón refresh existente.

Definition of Done:
	•	En iOS Safari y Chrome Android, en cualquier pantalla, el gesto pull-down NO recarga la página.
