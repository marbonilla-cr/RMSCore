5) REGLAS DIARIAS DE CÁLCULO (BASE LEGAL + REGLAS DE NEGOCIO)
Inputs diarios: schedule { scheduledStartAt, scheduledEndAt }, punches del día, user { dailyRate }, HR Config.

Múltiples punches mismo día:
- v1: usar primer clock_in y último clock_out.
- Auditar punches intermedios (audit/log).

Tardía (amonestación):
- late = clock_in > scheduledStartAt
- tardyMinutes = max(0, clock_in - scheduledStartAt)
- La tardía NO elimina pago por el tiempo trabajado real.

No show (amonestación):
- Si existe schedule para el día y NO hay clock_in => noShow = true, pago base = 0.

Worked time:
- workedStart = clock_in
- workedEnd:
  - Si clock_out MANUAL/REAL: workedEnd = clock_out
  - Si falta salida (AUTO por sistema): workedEnd = scheduledEndAt (o regla definida en HR Config si existe)
- workedMinutes = max(0, workedEnd - workedStart)

Horas extra (cumplimiento legal + regla negocio):
- Overtime NO se calcula por umbral fijo, se calcula por horario:
  - Si clock_out es MANUAL/REAL y scheduledEndAt existe:
      overtimeMinutes = max(0, clock_out - scheduledEndAt)
    Si clock_out es AUTO o clock_out <= scheduledEndAt:
      overtimeMinutes = 0
- normalMinutes = max(0, workedMinutes - overtimeMinutes)

Redondeos:
- Aplicar regla de redondeo definida en HR Config (si existe).
- Si no existe, usar minuto exacto sin redondeo. Documentar.

Pago:
- hourlyRate = dailyRate / jornadaOrdinariaHorasPorDia
- normalPay = (normalMinutes/60) * hourlyRate
- overtimePay = (overtimeMinutes/60) * hourlyRate * multiplicadorHoraExtra
  donde multiplicadorHoraExtra viene de HR Config; default legal común = 1.5 si no existe.
- baseTotalPayDay = normalPay + overtimePay

Turnos cruzando medianoche:
- Soportar con datetime real (18:00–02:00). No usar comparación de horas sin fecha.
