Objective

Replace the current hours-prorated service charge distribution with two simple report-only modes selectable in the payroll report UI:

1) BOLSA
For each day in range:
- Sum all assigned service amounts for valid waiters that day = poolDay
- Count unique waiter employee IDs with assigned service > 0 that day
- Each eligible waiter gets (poolDay / eligibleCount) * serviceRate
- Sum daily allocations across the selected date range

2) VENTA_MESERO
For each day in range:
- Each waiter gets (their own assigned service amount that day) * serviceRate
- Sum daily allocations across the selected date range

Important rules
- This is report logic only. Do not persist payroll results to DB.
- Remove all dependency on worked hours, punches, schedules, and computeDailyPayroll for service distribution.
- Exclude employeeId = 0 or null from both allocation modes.
- Exclude unassigned service from distribution; report it separately as serviceUnassignedTotal.
- If BOLSA mode has poolDay > 0 but zero eligible waiters, send that day’s pool to serviceUnassignedTotal and do not divide.
- Preserve response compatibility: return { result, serviceUnassignedTotal, allocationModeByDate }.
- allocationModeByDate should simply return the selected mode for each date in range.
