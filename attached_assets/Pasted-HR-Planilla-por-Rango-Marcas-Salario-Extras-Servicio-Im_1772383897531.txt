T003: Create payroll computation engine (server/payroll.ts) (FUNCIONES PURAS)
Crear server/payroll.ts con funciones puras (sin DB):
A) computeDailyPayroll({ punchesForDay, scheduleForDay, dailyRate, hrConfig }):
- hrConfig: overtimeDailyThresholdHours, overtimeMultiplier, latenessGraceMinutes
- hourlyRate = dailyRate / Number(hrConfig.overtimeDailyThresholdHours || 8)
- Multiple punches same day: first clockIn + last clockOut; hasMultiplePunches flag
- noShow:
  - si scheduleForDay existe AND !isDayOff (si existe ese flag en schedule) AND no hay clockIn => noShow=true; basePay=0
- Sin schedule:
  - si hay punch => calcular normally (no tardy/noShow)
- tardyMinutes:
  - si hay scheduleStart:
    - si clockIn > scheduleStart + grace => tardyMinutes = (clockIn - scheduleStart) (diferencia completa, NO restar gracia)
    - grace solo determina si late=true
  - si clockIn <= scheduleStart + grace => late=false, tardyMinutes=0
- Early arrival:
  - clockIn < scheduleStart => tardy=0, pago desde clockIn
- workedEnd:
  - si clockOutType=MANUAL => clockOut
  - si AUTO/missing => scheduledEndAt (autoCheckout=true)
- workedMinutes = max(0, workedEnd - clockIn) (minutos)
- overtimeMinutes:
  - SOLO si clockOutType=MANUAL y clockOut > scheduledEndAt => max(0, clockOut - scheduledEndAt)
  - si AUTO o clockOut <= scheduledEndAt => 0
- normalMinutes = workedMinutes - overtimeMinutes
- normalPay = round2((normalMinutes/60) * hourlyRate)
- overtimePay = round2((overtimeMinutes/60) * hourlyRate * Number(hrConfig.overtimeMultiplier || 1.5))
- baseTotalPayDay = round2(normalPay + overtimePay)
- Midnight-crossing:
  - usar datetime completos; scheduledEndAt ya debe ser fecha siguiente si cruza medianoche
- Return:
  {
    workedMinutes, normalMinutes, overtimeMinutes, tardyMinutes,
    normalPay, overtimePay, baseTotalPayDay,
    flags: { late, noShow, autoCheckout, midnightShift, hasMultiplePunches }
  }

B) computeServiceForDay({ businessDate, eligibleWaiters, servicePool, workedMinutesByEmployee }):
- servicePool = SUM(serviceAmount) del service_charge_ledger para businessDate
- weight(i)=workedMinutes(i)/totalWorkedMinutes
- servicePay(i)=round2(servicePool*weight(i))
- si totalWorkedMinutes=0 => todo 0
- Return: map employeeId -> servicePayDay

C) computeRangePayroll({ employees, schedulesMap, punchesMap, extrasMap, servicePoolMap, dateFrom, dateTo, hrConfig }):
- Iterar cada fecha del rango, y cada empleado.
- daily = computeDailyPayroll
- extrasDay = sum(extras del día)
- Servicio:
  - SOLO empleados con rol/permiso de WAITER (usar roles/perms existentes)
  - Calcular servicePayDay con computeServiceForDay usando eligibleWaiters del día
  - IMPORTANTE: el pool se usa por día (businessDate)
- Agregar dailyBreakdown por empleado:
  Fecha | worked | normal | overtime | tardy | basePay | extras | service | flags
- Agregados por empleado:
  daysScheduled, daysPresent, daysNoShow,
  totalNormalMin, totalOvertimeMin, totalLateMin, lateCount,
  normalPay, overtimePay, basePayTotal,
  extrasEarnings, extrasDeductions, extrasNet,
  servicePayTotal,
  grandTotalPay = round2(basePayTotal + extrasNet + servicePayTotal)
- Acceptance:
  - Pasa 8 casos base + extras + servicio prorrateo
