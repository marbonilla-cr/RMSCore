## CORRECCIГ“N 1 вЂ” `parseDateInTZ`: construir fechas con timezone CR

**Problema:** La funciГіn construye fechas con `setHours()` usando la
hora local del servidor. En Railway/Replit el servidor corre en UTC,
por lo que todos los horarios quedan desplazados 6 horas.

**Localizar** la funciГіn `parseDateInTZ` (lГ­neas ~151вЂ“156):
```typescript

  const [h, m] = timeStr.split(":").map(Number);
  const d = new Date(dateStr + "T00:00:00");
  d.setHours(h, m, 0, 0);
  return d;
}
```

**Reemplazar completamente por:**
```typescript
function parseDateInTZ(dateStr: string, timeStr: string): Date {
  // Construir con offset explГ­cito de Costa Rica (UTC-6)
  // Formato: "2026-03-05T08:00:00-06:00"
  return new Date(`${dateStr}T${timeStr}:00-06:00`);
}
```

**Verificar** que `getScheduleTimesForDate` sigue funcionando igual вЂ”
no necesita cambios porque sigue llamando a `parseDateInTZ`.

## CORRECCIГ“N 2 вЂ” `ordinaryEnd` y cГЎlculo de extras

**Problema:** El sistema calcula `ordinaryEnd = min(scheduledEnd, clockOut)`
y las extras como `clockOut - scheduledEnd`. Esto es incorrecto cuando
el empleado llega tarde: trabaja sus 8h exactas pero el sistema le
paga 7h ordinarias + 1h extra.

**La regla correcta (CГіdigo de Trabajo CR, Art. 136 y 139):**
- El tiempo ordinario se cuenta desde `paidStart` (que ya tiene el
  cap de llegada tarde).
- Las primeras `jornadaHoras` desde `paidStart` son ordinarias.
- Todo lo que supere `jornadaHoras` desde `paidStart` son extras.
- Solo aplica si `clockOutType === "MANUAL"` вЂ” los cierres automГЎticos
  (AUTO, AUTO_NO_SCHEDULE, GEO_AUTO) nunca generan extras.

**Localizar** el bloque que empieza en lГ­nea ~306:
```typescript
  const ordinaryEnd = scheduledEnd && lastClockOut.getTime() > scheduledEnd.getTime()
    ? scheduledEnd : lastClockOut;

  const normalMinutesRaw = Math.max(0, diffMinutes(paidStart, ordinaryEnd));

  let overtimeCalculatedMinutes = 0;
  if (scheduledEnd && lastClockOutType === "MANUAL" && lastClockOut.getTime() > scheduledEnd.getTime()) {
    overtimeCalculatedMinutes = diffMinutes(scheduledEnd, lastClockOut);
  }

  let overtimePaidMinutes = overtimeCalculatedMinutes;
  if (overtimeRequiresApproval && overtimeCalculatedMinutes > 0) {
    overtimePaidMinutes = 0;
    flags.push("OVERTIME_PENDIENTE_APROBACION");
  }
```

**Reemplazar completamente por:**
```typescript
  // ordinaryEnd = paidStart + jornadaHoras (ej: paidStart 9am + 8h = 5pm)
  // Las extras son todo lo que supere ese lГ­mite, solo si salida MANUAL
  const ordinaryEnd = new Date(paidStart.getTime() + jornadaHoras * 3600000);

  const normalMinutesRaw = Math.max(0, diffMinutes(paidStart,
    lastClockOut.getTime() < ordinaryEnd.getTime() ? lastClockOut : ordinaryEnd
  ));

  let overtimeCalculatedMinutes = 0;
  const isManualOut = lastClockOutType === "MANUAL";
  if (isManualOut && lastClockOut.getTime() > ordinaryEnd.getTime()) {
    overtimeCalculatedMinutes = diffMinutes(ordinaryEnd, lastClockOut);
  }

  // Las extras siempre se pagan si fueron trabajadas (CГіdigo de Trabajo CR Art. 139)
  // overtimeRequiresApproval solo activa el flag de visibilidad para el manager,
  // NUNCA bloquea el pago
  let overtimePaidMinutes = overtimeCalculatedMinutes;
  if (overtimeRequiresApproval && overtimeCalculatedMinutes > 0) {
    flags.push("OVERTIME_PENDIENTE_APROBACION");
    // NOTA: overtimePaidMinutes NO se pone en 0 вЂ” el pago es obligatorio por ley
  }
```

## CORRECCIГ“N 3 вЂ” DeducciГіn de almuerzo sobre total efectivo

**Problema:** El descuento de almuerzo se aplica sobre `normalMinutesRaw`
(solo horas ordinarias), cuando deberГ­a aplicarse sobre el total de
tiempo efectivo trabajado. AdemГЎs el umbral debe verificarse contra el
tiempo total, no solo ordinario.

**Regla de negocio:**
- Si tiempo efectivo total < 9h (540 min) в†’ sin descuento, el empleador absorbe
- Si tiempo efectivo total в‰Ґ 9h (540 min) в†’ descontar 60 min (30 almuerzo + 30 cafГ©)
- El descuento reduce primero las extras, luego las ordinarias si es necesario

**Localizar** el bloque del almuerzo (~lГ­neas 322вЂ“329):
```typescript
  let unpaidBreakMinutes = 0;
  if (breakDeductEnabled) {
    const payableWindow = normalMinutesRaw + overtimeCalculatedMinutes;
    if (payableWindow >= breakThresholdMin) {
      unpaidBreakMinutes = breakDeductMin;
    }
  }
  const normalMinutesPaid = Math.max(0, normalMinutesRaw - unpaidBreakMinutes);
```

**Reemplazar completamente por:**
```typescript
  let unpaidBreakMinutes = 0;
  let normalMinutesPaid = normalMinutesRaw;
  let overtimePaidAfterBreak = overtimePaidMinutes;

  if (breakDeductEnabled) {
    // Umbral sobre tiempo efectivo total (ordinario + extra calculado)
    const totalEffectiveMinutes = normalMinutesRaw + overtimeCalculatedMinutes;
    if (totalEffectiveMinutes >= breakThresholdMin) {
      unpaidBreakMinutes = breakDeductMin;
      // El descuento se aplica primero sobre las extras, luego sobre ordinarias
      const deductFromOvertime = Math.min(unpaidBreakMinutes, overtimePaidAfterBreak);
      overtimePaidAfterBreak = Math.max(0, overtimePaidAfterBreak - deductFromOvertime);
      const remaining = unpaidBreakMinutes - deductFromOvertime;
      normalMinutesPaid = Math.max(0, normalMinutesRaw - remaining);
    }
  }

  // Usar overtimePaidAfterBreak para los cГЎlculos de pago
  overtimePaidMinutes = overtimePaidAfterBreak;
```
