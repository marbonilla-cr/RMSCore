IMPORTANTE:
- Por ahora los stocks se manejan en AP base (g/ml/unit) y el Conversion Builder calcula EP porciones para costeo.
- NO deducción automática por venta (FULL futuro).

CALCULOS:
Usar unidades base:
- peso => gramos
- volumen => ml
- unidad => unit

Para 1 unidad AP:
usable_qty = ap_qty_base * (1 - trim_waste_pct/100) * cook_factor * (1 - extra_loss_pct/100)

Por output:
portions_generated = usable_qty / portion_size (si g/ml)
Para unit/portion:
- permitir portion_size como 1.0 o 0.33 etc.
- si compra por unit y requiere g -> exigir unit_weight_g para conversion a g.

Costo:
cost_per_portion = purchase_cost_for_ap_qty / portions_generated
Mostrar también cost_per_base_unit (por g/ml/unit).
