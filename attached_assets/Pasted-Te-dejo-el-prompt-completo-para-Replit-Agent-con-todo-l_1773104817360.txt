RMS — Tenant Bootstrap Data Loader

Deterministic v1 (No AI)

Objective

Build a Tenant Bootstrap Data Loader that allows a new tenant to upload a Master Excel file containing initial operational data.

The system must:
	1.	Accept an Excel file
	2.	Parse sheets using a deterministic column alias mapper
	3.	Load rows into staging ledger tables
	4.	Provide an inline editor for review
	5.	Validate referential integrity
	6.	Execute a transactional import
	7.	Run an automatic system check
	8.	Mark tenant ready for operation

This version does not use AI.
Parsing uses deterministic alias mapping.

AI-assisted parsing can be added in a future version.

⸻

Architectural Principles
	•	Runs inside the tenant scope
	•	Never writes directly to production tables
	•	Uses ledger staging tables
	•	Import must be transactional
	•	A system check agent validates the environment

COLUMN ALIAS MAP

Parser must normalize column names.

Example alias map:

product_name → ["product","producto","name","nombre"]
price → ["price","precio","costo"]
category → ["category","categoria"]
employee_name → ["employee","empleado","name"]
tax_name → ["tax","impuesto"]
group_name → ["group","grupo"]
modifier_name → ["modifier","modificador"]

Aliases are case-insensitive.

TASKS

⸻

T001 — Create staging migration

Create staging schema and install Excel parser.

Tasks:
	•	Create migration 0004_data_loader_staging.sql
	•	Install xlsx (SheetJS)
	•	Ensure migration runs at startup

Acceptance Criteria:
	•	Tables exist
	•	Migration executes without errors
	•	Excel library loads

Definition of Done:
	•	Migration committed
	•	Tables visible in DB
	•	Server starts successfully

⸻

T002 — Excel parser service

Create parser:

server/data-loader/excel-parser.ts

Responsibilities:
	•	Read workbook
	•	Detect sheets
	•	Convert rows to JSON
	•	Normalize headers via alias map

Acceptance:
	•	Parser reads all sheets
	•	Rows converted to JSON

DoD:
	•	Parser returns structured output
	•	No crash on missing sheets

⸻

T003 — Column alias mapper

Create:

server/data-loader/column-aliases.ts

Responsibilities:
	•	Normalize column names
	•	Map alias → canonical field

Acceptance:
	•	Aliases correctly resolved
	•	Unknown columns ignored

DoD:
	•	Alias mapping tested

⸻

T004 — Staging writer

Insert rows into staging table.

File:

server/data-loader/staging-writer.ts

Acceptance:
	•	Rows stored with sheet_name
	•	session_id linked

DoD:
	•	All rows visible in staging

⸻

T005 — Inline ledger editor API

Create endpoints:

GET /data-loader/session/:id
PATCH /data-loader/row/:id

Acceptance:
	•	Client can edit row data

DoD:
	•	Edits persist in staging

⸻

T006 — Validation engine

Validation rules:

Products.category must exist in Categories
Products.tax_name must exist in Taxes
Modifiers.group_name must exist

Acceptance:
	•	Invalid rows flagged
	•	Errors stored in validation_errors

DoD:
	•	validation_status updated

⸻

T008 — System check

Verify tenant is operational.

Checks:

business exists
tax exists
payment methods exist
at least one employee exists

Acceptance:

DoD:
	•	Tenant flagged ready
