# Prompt — Reconstruir página QR del cliente (`client/src/pages/qr-client.tsx`)

## Contexto
Estás trabajando en un sistema de restaurante (PWA, React + Vite, Tailwind CSS, wouter).  
El archivo a reemplazar es `client/src/pages/qr-client.tsx`.  
La ruta es `/qr/:tableCode` — **pública, sin autenticación**.

**No toques ningún otro archivo.** Solo reescribís `qr-client.tsx`.

---

## Objetivo
Reemplazar el flujo QR actual (7+ pasos, selección explícita de subcuenta, selección de modo Easy/Standard) por un flujo simplificado de **4 pasos lineales**, con un popup de producto que incluye modificadores, cantidad y nota antes de agregar al carrito.

---

## Flujo de pantallas

```
S0: Bienvenida  →  S1: Nombre  →  S2: Menú  →  S3: Revisión  →  S4: Enviado
```

La selección de subcuenta desaparece de la vista del cliente. El backend la asigna automáticamente por nombre: si ya existe una subcuenta con ese nombre en la mesa, se reutiliza; si no, se crea una nueva. Esto ya lo maneja `POST /api/qr/:tableCode/submit` — solo asegurate de enviar `customerName` en cada ítem.

---

## APIs a usar (sin cambios)

| Método | Endpoint | Uso |
|--------|----------|-----|
| GET | `/api/qr/:tableCode/info` | Nombre de mesa, maxSubaccounts |
| GET | `/api/qr/:tableCode/menu` | Productos activos + categorías TOP |
| GET | `/api/products/:id/modifiers` | Grupos de modificadores del producto |
| POST | `/api/qr/:tableCode/submit` | Enviar pedido |

### Estructura del submit
```typescript
POST /api/qr/:tableCode/submit
{
  customerName: string,           // nombre ingresado en S1
  items: Array<{
    productId: number,
    qty: number,
    notes: string,                // nota libre de cocina
    modifiers: Array<{            // opciones seleccionadas
      modGroupId: number,
      optionId: number
    }>
  }>
}
```

### Lógica del modal

```typescript
// Al abrir: fetch modificadores
const { data: groups } = useQuery({
  queryKey: [`/api/products/${product.id}/modifiers`],
})
// groups: Array<{ id, name, required, multiSelect, options: [{id, name, priceDelta}] }>

// Estado interno
const [qty,   setQty]   = useState(existing?.qty  ?? 1)
const [note,  setNote]  = useState(existing?.note ?? "")
const [sel,   setSel]   = useState<Record<string, number[]>>(
  existing?.selections ?? {}
)
const [tried, setTried] = useState(false)

// Precio dinámico
const modPrice = groups?.flatMap(g => g.options
  .filter(o => sel[g.id]?.includes(o.id))
  .map(o => o.priceDelta)
).reduce((a,b) => a+b, 0) ?? 0
const unitPrice = product.price + modPrice

// Validación
const missingRequired = groups?.filter(
  g => g.required && (!sel[g.id] || sel[g.id].length === 0)
) ?? []

// Toggle opción
function toggle(groupId: number, optionId: number, multi: boolean) {
  setSel(prev => {
    const cur = prev[groupId] ?? []
    if (multi) {
      return { ...prev, [groupId]: cur.includes(optionId)
        ? cur.filter(x => x !== optionId)
        : [...cur, optionId] }
    }
    return { ...prev, [groupId]: cur.includes(optionId) ? [] : [optionId] }
  })
}

// Al tocar "Pedir"
function handleAdd() {
  setTried(true)
  if (missingRequired.length > 0) return
  // construir array de modifiers para el submit
  const modifiers = Object.entries(sel).flatMap(([groupId, optIds]) =>
    optIds.map(optionId => ({ modGroupId: Number(groupId), optionId }))
  )
  onAdd(product, qty, note, sel, unitPrice)
  onClose()
}
```

## Manejo de errores y estados de carga

- Mientras carga `info` o `menu`: spinner centrado con Loader2
- Error en fetch: toast destructivo, no rompe la página
- Error en submit: toast destructivo, botón "Confirmar" vuelve a habilitarse
- Durante el submit: botón muestra "⏳ Enviando..." y `disabled`

---

## Comportamiento del carrito

- El carrito vive en `useState` — no en localStorage para esta página
- `unitPrice` del ítem = `product.price + suma(priceDelta de opciones seleccionadas)`
- Si el cliente toca un producto ya agregado → el modal se abre pre-llenado con qty, note y selections actuales. El botón dice "Actualizar". Al confirmar, reemplaza el ítem existente.
- Si qty baja a 0 en S3 → ítem eliminado del carrito. Si carrito queda vacío → `go(2)`
