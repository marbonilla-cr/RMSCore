T001: Planilla servicio — elegibilidad por ledger, no por rol
FILES: server/payroll.ts, server/routes.ts

A) server/payroll.ts — computeServiceForRange
1) ELIMINAR filtro por rol:
   eliminar: if (emp.role !== "WAITER" && emp.role !== "SALONERO") continue

2) Construir eligibleIdsByDate desde serviceLedgerByEmployee:
   eligibleIdsByDate[dateStr] = new Set(
     Object.keys(serviceLedgerByEmployee[dateStr] || {})
       .map((k) => Number(k))
       .filter((id) => Number.isFinite(id) && id > 0)
   )
   (NO incluir 0)

3) Para cada dateStr del rango:
   - ledgerDay = serviceLedgerByEmployee[dateStr] || {}
   - unassignedDay = Number(ledgerDay[0] || 0)
   - poolDay = sum(Object.values(ledgerDay).map(Number))  // incluye 0 si existe
     (NO depender de servicePoolMap aquí, para evitar desalineación)

   Reglas:
   a) Si poolDay <= 0:
      allocationModeByDate[dateStr] = "UNASSIGNED"
      continue

   b) eligibleIds = eligibleIdsByDate[dateStr]
      Si eligibleIds.size === 0:
        serviceUnassignedTotal += poolDay
        allocationModeByDate[dateStr] = "UNASSIGNED"
        continue

   c) Intento HOURS_PRORATED:
      - payablePool = poolDay - unassignedDay
      - Construir eligiblesConMin = eligibleIds con paidWorkedMinutes(empId,dateStr) > 0
      - sumMin = sum(paidWorkedMinutes de eligiblesConMin)
      - Si payablePool > 0 y sumMin > 0:
          para empId en eligiblesConMin:
            weight = paidMin(empId)/sumMin
            result[empId] += round2(payablePool * weight)
          serviceUnassignedTotal += unassignedDay
          allocationModeByDate[dateStr] = "HOURS_PRORATED"
          continue

   d) Fallback LEDGER_DIRECT (si no hay punches o sumMin==0):
      - para cada empId en eligibleIds:
          result[empId] += Number(ledgerDay[empId] || 0)
      - serviceUnassignedTotal += unassignedDay
      - allocationModeByDate[dateStr] = "LEDGER_DIRECT"

4) Retornar:
   { result, serviceUnassignedTotal, allocationModeByDate }

T003: Flag “Operó como Mesero” (derivado, sin cambiar role)
FILES: server/routes.ts, client/src/pages/hr/reports.tsx

Backend (routes.ts payroll-report):
- Para el rango de planilla:
  waiterIdsFromSales =
    SELECT DISTINCT responsible_waiter_id
    FROM sales_ledger_items
    WHERE business_date BETWEEN dateFrom AND dateTo

- Para el rango de servicio:
  waiterIdsFromService =
    SELECT DISTINCT responsible_waiter_employee_id
    FROM service_charge_ledger
    WHERE business_date BETWEEN serviceFrom AND serviceTo

- operationalWaiterIds = union(waiterIdsFromSales, waiterIdsFromService).filter(id > 0)
- Para cada empleado en response:
  operatedAsWaiter = operationalWaiterIds.has(employeeId)

Frontend (reports.tsx):
