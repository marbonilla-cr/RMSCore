## Prompt para Replit Agent

Hay un bug en producción: el tenant de La Antigua tiene `slug = 'rest-la-antigua'` pero la URL de producción es `rms.restlaantigua.com`, entonces el middleware busca `slug = 'rms'` y no encuentra nada → 500 en todos los endpoints.

**CAMBIO 1 — Corregir el slug en Railway.**

Crear y ejecutar `scripts/fix-slug.ts`:

```typescript
import { Pool } from 'pg';
const pool = new Pool({ connectionString: process.env.DATABASE_URL });

const before = await pool.query(`SELECT id, slug, schema_name, is_active FROM public.tenants`);
console.log('BEFORE:', JSON.stringify(before.rows, null, 2));

await pool.query(`UPDATE public.tenants SET slug = 'rms' WHERE schema_name = 'public'`);

const after = await pool.query(`SELECT id, slug, schema_name, is_active FROM public.tenants`);
console.log('AFTER:', JSON.stringify(after.rows, null, 2));

await pool.end();
```

Mostrar el output completo. Luego eliminar el archivo.

**CAMBIO 2 — Hacer el middleware más robusto** en `server/middleware/tenant.ts`.

Buscar el bloque catch al final y reemplazar:
```typescript
} catch (err: any) {
  console.error("[tenant-middleware] Error:", err.message);
  req.tenantSchema = process.env.TENANT_SCHEMA || "public";
```

Por:
```typescript
} catch (err: any) {
  console.error("[tenant-middleware] Error:", err.message, "host:", req.hostname);
  req.tenantSchema = process.env.TENANT_SCHEMA || "public";
```

Y buscar el bloque donde no encuentra el tenant:
```typescript
if (!result.rows.length) return res.status(404).json({ message: "Restaurante no encontrado." });
```

Reemplazar por:
```typescript
if (!result.rows.length) {
  console.error("[tenant-middleware] Slug no encontrado:", slug, "host:", req.hostname);
  return res.status(404).json({ message: "Restaurante no encontrado.", slug });
}
```
