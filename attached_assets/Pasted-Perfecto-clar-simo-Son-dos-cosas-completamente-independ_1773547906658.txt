Perfecto, clarísimo. Son dos cosas completamente independientes:

**Código 3 caracteres** — trazabilidad operativa de la transacción dentro del tenant. Solo sirve para pagar y recoger. No tiene relación con loyalty.

**Loyalty** — se acumula automáticamente al cobrar si el cliente está vinculado a la orden. El cajero puede vincular o no — es opcional.

Entonces el flujo es:

```
Orden creada en tenant X → genera código "A7K" (único en ese tenant en ese momento)
                         → cliente paga con código "A7K"  
                         → si tiene cuenta loyalty → puntos acumulados automáticamente
                         → cocina llama "A7K" → cliente retira
```

El código es **por tenant, secuencial rotativo dentro del día, aleatorio entre tenants simultáneos**.

### PARTE B — Generador de código en backend

Crear `server/utils/transaction-code.ts`:

```typescript
import { eq, and, sql } from "drizzle-orm";
import { orders } from "@shared/schema";

const CHARS = "ABCDEFGHJKLMNPQRSTUVWXYZ23456789"; // sin O,0,I,1 para evitar confusión

export async function generateTransactionCode(
  db: any,
  businessDate: string
): Promise<string> {
  // Obtener todos los códigos usados hoy
  const usedCodes = await db
    .select({ code: orders.transactionCode })
    .from(orders)
    .where(
      and(
        eq(orders.businessDate, businessDate),
        sql`${orders.transactionCode} IS NOT NULL`
      )
    );

  const used = new Set(usedCodes.map((r: any) => r.code));

  // Generar código aleatorio que no esté en uso
  let attempts = 0;
  while (attempts < 1000) {
    const code =
      CHARS[Math.floor(Math.random() * CHARS.length)] +
      CHARS[Math.floor(Math.random() * CHARS.length)] +
      CHARS[Math.floor(Math.random() * CHARS.length)];
    if (!used.has(code)) return code;
    attempts++;
  }
  // Fallback: timestamp si se agotaron combinaciones (prácticamente imposible)
  return Date.now().toString(36).slice(-3).toUpperCase();
}
```
