RMS — AI Data Loader (Deterministic v1)

Tenant Bootstrap Import System

Objetivo

Construir un Data Loader para bootstrap de tenants que permita cargar un Excel maestro, mapear columnas mediante un sistema determinístico con alias, organizar la información en tablas de staging tipo ledger, permitir edición antes de importar, validar los datos y ejecutar un import transaccional seguro hacia las tablas productivas.

Esta primera versión no usa IA.
El parser será determinístico con alias flexibles.

La capa de IA podrá agregarse posteriormente.

⸻

Principios arquitectónicos
	1.	El Data Loader vive dentro del tenant, no en el core global.
	2.	Nunca escribe directamente en tablas productivas.
	3.	Usa staging tables tipo ledger.
	4.	El import final es transaccional.
	5.	El sistema ejecuta un system check automático antes de activar el tenant.

HU-DL-001

Subida del Excel

Historia

Como administrador de tenant
quiero subir un archivo Excel
para iniciar el proceso de importación de datos.

Implementación

Endpoint:

POST /tenant/data-loader/upload

Aceptar:

.xlsx

Guardar archivo en almacenamiento temporal del tenant.

Registrar sesión de import.

Tabla

data_loader_sessions

id
tenant_id
file_path
status
created_at

Status posibles:

uploaded
parsed
staged
validated
imported
failed


⸻

Criterios de aceptación
	1.	El sistema acepta archivos .xlsx.
	2.	Si el archivo no es Excel, el sistema rechaza la carga.
	3.	Se crea una sesión de import.
	4.	El archivo se asocia al tenantId.

⸻

Definition of Done
	•	endpoint funcional
	•	archivo almacenado
	•	sesión registrada
	•	manejo de errores implementado

HU-DL-002

Parser determinístico del Excel

Historia

Como sistema
quiero leer el Excel y detectar las tablas
para convertirlo en estructuras normalizadas.

⸻

Implementación

Servicio:

services/excelParser.ts

Librería recomendada:

xlsx

Leer todas las hojas del Excel.

Cada hoja se convierte en JSON.

Ejemplo:

[
 { "nombre": "Hamburguesa", "precio": 8.5 },
 { "nombre": "Pizza", "precio": 10 }
]


⸻

Criterios de aceptación
	1.	Todas las hojas del Excel se convierten a JSON.
	2.	El parser detecta encabezados.
	3.	Filas vacías se ignoran.

⸻

Definition of Done
	•	parser estable
	•	manejo de archivos grandes
	•	logs de parsing

HU-DL-007

Import transaccional

Historia

Como sistema
quiero mover datos de staging a tablas productivas
de forma segura.

⸻

Implementación

Servicio:

services/importExecutor.ts

Proceso:

BEGIN TRANSACTION
insert business
insert taxes
insert payment methods
insert categories
insert products
insert modifiers
insert employees
COMMIT

Si falla:

ROLLBACK


⸻

Criterios de aceptación
	1.	El import es atómico.
	2.	Si falla un paso, nada se escribe.
	3.	Logs detallados.

⸻

Definition of Done
