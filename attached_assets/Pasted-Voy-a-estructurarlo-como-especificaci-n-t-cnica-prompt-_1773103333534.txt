RMS — AI Data Loader

Tenant Bootstrap Import System

1. Objetivo

Construir un AI Data Loader que permita a un nuevo tenant cargar toda la información inicial del sistema mediante un archivo Excel único.

El sistema debe:
	1.	aceptar un Excel maestro
	2.	usar IA para interpretar y organizar los datos
	3.	cargar la información en un ledger intermedio
	4.	permitir edición directa por el usuario
	5.	validar los datos
	6.	ejecutar un import transaccional
	7.	realizar verificación automática del sistema
	8.	activar el tenant para operación

El AI Data Loader no escribe directamente en tablas productivas.
Todo pasa primero por un entorno intermedio dentro del tenant.

⸻

2. Arquitectura

El módulo vive dentro del tenant, no en el core global del RMS.

Arquitectura:

6. AI Parser

El parser usa IA para:
	•	detectar columnas
	•	mapear nombres
	•	corregir errores comunes
	•	normalizar estructuras

Ejemplo:

"precio"
"price"
"cost"

→ product_price

"categoria"
"category"
"grupo"

→ category_name

⸻

7. UI Ledger Inline

Después del parseo se muestra una interfaz tipo spreadsheet.

Cada hoja aparece como tabla editable.

Ejemplo:

Products
-------------------------------------------
Name       | Category | Price | Tax | SKU
-------------------------------------------
Burger     | Food     | 8.50  | 13% | 1001
Pizza      | Food     | 9.50  | 13% | 1002

El usuario puede:
	•	editar campos
	•	agregar filas
	•	eliminar filas
	•	corregir errores

10. System Check Agent

Después del import el sistema entra en modo verificación automática.

Checks:

employees count > 0
tax exists
payment method exists
tables exist

Checks de integridad:

modifiers.group_id valid
product_modifier_relations valid

Si todo pasa:

13. Prompt listo para el agente de Replit

Este prompt puede usarse directamente para que el agente construya el módulo.

Build a new RMS module called AI Data Loader.

Requirements:

The module must run inside a tenant environment and never write directly to production tables.

The system must accept a single Excel file containing multiple sheets.

Sheets may include:

Business
Taxes
PaymentMethods
Employees
MenuCategories
Products
ModifierGroups
Modifiers
ProductModifiers
Tables
PrinterRoutes
KDSStations
InventoryItems
Recipes
HRConfig

If a sheet is missing it should be ignored.

Step 1
Create staging tables for each entity with structure:

id
tenant_id
row_source
data_json
validation_status
validation_errors
created_at
updated_at

Step 2
Build an Excel parser that reads each sheet and stores rows into the corresponding staging table.

Step 3
Create an AI normalization layer that maps flexible column names into the RMS schema.

Step 4
Create a UI component that displays staging data as editable tables (spreadsheet style).

Users must be able to edit fields inline before import.

Step 5
Create a validation engine that validates required fields and relationships.

Step 6
Create a transactional import executor that moves validated staging data into production tables.

Step 7
After import run a system verification agent that checks integrity and readiness.

Step 8
If all checks pass mark tenant as ACTIVE.

Ensure all operations are tenant isolated.

Do not modify core RMS schemas.
