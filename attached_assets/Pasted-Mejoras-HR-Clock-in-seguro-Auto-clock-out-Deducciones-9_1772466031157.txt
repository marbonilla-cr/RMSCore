T001: Clock-in sin horario requiere doble confirmación
FILES: server/routes.ts, client/src/pages/pin-login.tsx, client/src/pages/hr/mi-turno.tsx

BACKEND (server/routes.ts)
En POST /api/auth/pin-clock (línea ~630) y POST /api/hr/clock-in (línea ~4909):
1) Determinar si el empleado tiene horario programado para “hoy”.
   - “No hay horario” significa:
     a) No existe schedule del día, o
     b) Existe pero es día libre / sin horas (si hay isDayOff o equivalente), o
     c) No tiene scheduledStartAt/scheduledEndAt válidos.
2) Si NO hay horario y request NO trae confirmNoSchedule: true:
   - Responder 200 con:
     { requireConfirm: true, message: "No tiene horario hoy. Ingrese su código nuevamente para confirmar." }
   - NO crear punch.
3) Si NO hay horario y request trae confirmNoSchedule: true:
   - Permitir clock-in normal.
   - Registrar audit_event: action="CLOCK_IN_NO_SCHEDULE_CONFIRMED" con employeeId, date, metadata.
4) Si SÍ hay horario:
   - Clock-in normal (sin confirmación).

FRONTEND
- pin-login.tsx y mi-turno.tsx:
  - Si respuesta contiene requireConfirm=true:
    - Mostrar modal/dialog con el message.
    - Si usuario confirma: reenviar el mismo request con confirmNoSchedule=true.
    - Si cancela: no reenviar (opcional: NO auditar cancelación).
  - Asegurar que confirmNoSchedule solo aplica a ese intento; no persistir globalmente.

ACCEPTANCE
- Clock-in sin horario pide confirmación; sin confirmación no marca.
- Con confirmNoSchedule=true marca + audit_event.
- Con horario funciona normal.

T003: Deducción automática almuerzo/café (>= 9h) sin tocar overtime
FILES: server/payroll.ts, client/src/pages/hr/reports.tsx

En server/payroll.ts computeDailyPayroll (o función equivalente usada por reportes):
PASOS (orden obligatorio):
1) Calcular workedMinutes (RAW) como hoy (clockIn->workedEnd). workedEnd:
   - MANUAL: clockOutAt
   - AUTO/missing: scheduledEndAt (si existe); si no existe, asumir que el job T002 lo cerrará (para reporte, si aún está abierto, tratar como “hasta now” solo si ya existe esa conducta; si no, dejar 0 y flag).
2) Calcular overtimeMinutes por horario (NO umbral):
   - SOLO si clockOutType=MANUAL y scheduledEndAt existe:
     overtimeMinutes = max(0, clockOutAt - scheduledEndAt)
   - si AUTO o clockOutAt <= scheduledEndAt => 0
3) normalMinutesRaw = max(0, workedMinutes - overtimeMinutes)
4) unpaidBreakMinutes = (workedMinutes >= 540) ? 60 : 0
5) Aplicar descuento SOLO al ordinario:
   - normalMinutesPaid = max(0, normalMinutesRaw - unpaidBreakMinutes)
   - overtimeMinutesPaid = overtimeMinutes (NO se reduce)
6) Definir paidWorkedMinutes = normalMinutesPaid + overtimeMinutesPaid (para servicio y UI).
7) Calcular pagos:
   - hourlyRate = dailyRate / Number(hrConfig.overtimeDailyThresholdHours || 8)
   - multiplier = Number(hrConfig.overtimeMultiplier || 1.5)
   - normalPay = round2((normalMinutesPaid/60) * hourlyRate)
   - overtimePay = round2((overtimeMinutesPaid/60) * hourlyRate * multiplier)
8) Exponer unpaidBreakMinutes y paidWorkedMinutes en DailyPayrollResult.
9) Actualizar tipos/DTOS:
   - DailyBreakdownEntry: incluir unpaidBreakMinutes, paidWorkedMinutes
   - EmployeePayrollResult: incluir totalUnpaidBreakMinutes (acumulado)
10) Actualizar computeRangePayroll:
   - acumular totalUnpaidBreakMinutes
11) Actualizar computeServiceForDay:
   - usar paidWorkedMinutes como peso de prorrateo (no raw).

UI (client/src/pages/hr/reports.tsx):
- Agregar columna "Desc." (minutos) en tabla principal y breakdown diario.
- Mostrar como HH:mm (ej: 60 => "01:00").
- Totales incluyen suma de descuentos.
- Asegurar que “Servicio” usa paidWorkedMinutes (para que descansos no pagados reduzcan peso).

ACCEPTANCE

- workedMinutes < 540 => unpaidBreakMinutes=0
- Deducción afecta solo ordinario, overtime intacto.
- UI muestra descuento y totales correctos.
