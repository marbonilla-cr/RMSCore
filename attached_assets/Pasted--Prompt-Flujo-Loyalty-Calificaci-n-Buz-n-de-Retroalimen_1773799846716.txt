## Prompt — Flujo Loyalty + Calificación + Buzón de Retroalimentación

### ¿Qué queremos lograr?

Cuando un cliente termina de ser atendido — sea en una orden QR de mesa o en despacho en mostrador — queremos aprovechar ese momento de satisfacción para tres cosas simultáneas:

1. **Mostrarle sus puntos de loyalty** — el cliente ve inmediatamente cuánto ganó por su visita, reforzando el valor del programa
2. **Pedirle una calificación** — mientras tiene el teléfono en la mano y la experiencia fresca
3. **Dirigir las buenas experiencias a Google** — las calificaciones de 4-5 estrellas van a Google Business donde otros clientes las ven; las malas van directo a gerencia de forma privada

El resultado esperado es un ciclo virtuoso: cliente satisfecho → ve sus puntos → califica positivamente → reseña pública en Google → más clientes → más ventas → más puntos para clientes. Las malas experiencias van a gerencia antes de llegar a Google, protegiendo la reputación del restaurante.

**Este flujo no requiere ninguna acción del personal** — se dispara automáticamente cuando el KDS marca el ticket como terminado.

---

### Flujo completo esperado

```
KDS → "Ticket Terminado"
  → WebSocket event inmediato → teléfono del cliente
  → Pantalla 1: "¡Tu orden está lista! 🏆 +50 puntos ganados"
    → [Ver mis puntos →] abre loyalty.rmscore.app
    → [Calificarnos ⭐] avanza a pantalla 2
  → Pantalla 2: "¿Cómo estuvo tu experiencia?" ⭐⭐⭐⭐⭐
    → 4-5 estrellas → botón "Calificar en Google" → Google Business
    → 1-3 estrellas → mensaje privado a gerencia + email
  → Pantalla 3: "¡Gracias! Hasta la próxima 👋"
```

---

### ¿Por qué no funciona hoy?

El diagnóstico confirmó exactamente qué falta:

1. **El backend SÍ emite `dispatch_ready`** cuando despacho termina — pero el cliente no escucha ese evento
2. **No existe el evento `order_review_ready`** que unifique ambos flujos (QR y despacho) en un solo trigger
3. **No existe listener WebSocket** en `qr-client.tsx` para ninguno de estos eventos
4. **No existe tabla** para guardar calificaciones negativas (buzón interno)

Son exactamente 4 piezas que faltan. Nada más.

**Endpoint para guardar la calificación:**

```typescript
app.post("/api/qr/:tableCode/submit-review", async (req, res) => {
  try {
    const { orderId, rating, comment, customerName } = req.body;
    if (!orderId || !rating) {
      return res.status(400).json({ message: "orderId y rating requeridos" });
    }

    const config = await getBusinessConfig(req.db);
    const businessDate = await getBusinessDate(req.tenantSchema);

    // Idempotente — no crear duplicados
    const existing = await req.db.select()
      .from(feedbackMessages)
      .where(eq(feedbackMessages.orderId, orderId));
    if (existing.length > 0) {
      return res.status(409).json({ message: "Ya existe una calificación para esta orden" });
    }

    const [feedback] = await req.db.insert(feedbackMessages).values({
      orderId,
      rating,
      comment: comment || null,
      customerName: customerName || null,
      isRead: false,
      businessDate,
    }).returning();

    // Calificación baja → email a gerencia (silencioso, no falla el request)
    if (rating <= 3 && config?.email) {
      try {
        const { sendEmail } = await import("./services/email-service");
        await sendEmail({
          to: config.email,
          subject: `⚠️ Calificación baja: ${rating}/5 estrellas`,
          html: `
            <h2>Calificación recibida: ${"⭐".repeat(rating)}</h2>
            <p><strong>Orden:</strong> #${orderId}</p>
            <p><strong>Cliente:</strong> ${customerName || "Anónimo"}</p>
            <p><strong>Comentario:</strong> ${comment || "Sin comentario"}</p>
            <p><strong>Fecha:</strong> ${businessDate}</p>
          `,
        });
      } catch (emailErr) {
        console.error("[Review] Error enviando email a gerencia:", emailErr);
      }
    }

    // Notificar dashboard en tiempo real
    broadcast("feedback_received", { rating, orderId, isLow: rating <= 3 });

    res.json({
      success: true,
      reviewPoints: config?.reviewPoints || 50,
      // Solo retornar URL de Google si la calificación es positiva
      googleMapsReviewUrl: rating >= 4 ? (config?.googleMapsReviewUrl || null) : null,
    });
  } catch (err: any) {
    res.status(500).json({ message: err.message });
  }
});
```

**Endpoints del buzón:**

```typescript
// Listar mensajes
app.get("/api/admin/feedback", requireRole("MANAGER"), async (req, res) => {
  try {
    const messages = await req.db.select()
      .from(feedbackMessages)
      .orderBy(desc(feedbackMessages.createdAt))
      .limit(100);
    res.json(messages);
  } catch (err: any) {
    res.status(500).json({ message: err.message });
  }
});

// Contar no leídos
app.get("/api/admin/feedback/unread-count", requireRole("MANAGER"), async (req, res) => {
  try {
    const [result] = await req.db.select({
      count: sql<number>`count(*)::int`
    }).from(feedbackMessages)
      .where(eq(feedbackMessages.isRead, false));
    res.json({ count: result?.count || 0 });
  } catch (err: any) {
    res.status(500).json({ message: err.message });
  }
});

// Marcar como leído
app.patch("/api/admin/feedback/:id/read", requireRole("MANAGER"), async (req, res) => {
  try {
    const [updated] = await req.db.update(feedbackMessages)
      .set({ isRead: true })
      .where(eq(feedbackMessages.id, parseInt(req.params.id)))
      .returning();
    res.json(updated);
  } catch (err: any) {
    res.status(500).json({ message: err.message });
  }
});

// Marcar todos como leídos
app.post("/api/admin/feedback/mark-all-read", requireRole("MANAGER"), async (req, res) => {
  try {
    await req.db.update(feedbackMessages)
      .set({ isRead: true })
      .where(eq(feedbackMessages.isRead, false));
    res.json({ success: true });
  } catch (err: any) {
    res.status(500).json({ message: err.message });
  }
});
```
