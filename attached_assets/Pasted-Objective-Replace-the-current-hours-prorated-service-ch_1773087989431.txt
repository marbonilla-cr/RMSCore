Objective

Replace the current hours-prorated service charge distribution with two simple report-only modes selectable in the payroll report UI.

The two modes are:

1) BOLSA
For each day in the selected range:
- Sum all assigned service for valid waiters = poolDay
- Count eligible waiters for that day
- Each eligible waiter gets an equal share of that pool
- Then apply the service distribution percentage configured in the HR config module
- Sum all daily allocations across the selected date range

Formula:
distributedAmountPerWaiterPerDay = (poolDay / eligibleCount) * serviceDistributionPctFromHrConfig

2) VENTA_MESERO
For each day in the selected range:
- Each waiter gets their own assigned service for that day
- Then apply the service distribution percentage configured in the HR config module
- Sum all daily allocations across the selected date range

Formula:
distributedAmountPerWaiterPerDay = assignedServiceDay[empId] * serviceDistributionPctFromHrConfig

Important rules

- This is report logic only. Do not persist payroll results to DB.
- Remove all dependency on worked hours, punches, schedules, and computeDailyPayroll for service distribution.
- Valid waiters = employeeId > 0 with assigned service > 0 for that day.
- Exclude employeeId = 0 or null from both allocation modes.
- Exclude unassigned service from distribution; report it separately as serviceUnassignedTotal.
- If BOLSA mode has poolDay > 0 but zero eligible waiters, send that day’s pool to serviceUnassignedTotal.
- Preserve response compatibility: return { result, serviceUnassignedTotal, allocationModeByDate }.
- allocationModeByDate should map every date in the range to the selected mode only.
- Use the existing assigned service-by-employee/day source already used by payroll reporting. Do not introduce a new service source.
- The percentage applied to service distribution must always come from the existing HR config module.
- Do not hardcode the percentage.
- Do not ask the user to manually enter a percentage in the payroll report UI.
- The selected mode (BOLSA or VENTA_MESERO) only changes the distribution base, not the percentage source.
- Keep using the current HR configuration source already defined today for service distribution.
- The report must display the exact percentage used from HR config for auditability.

- allocationModeByDate = map of each date to the selected mode
